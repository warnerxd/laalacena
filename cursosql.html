INICIO:Bases de datos relacionales

    * Una base de datosrelacional es la recopilacion de elementos de datos
      con relaciones predefinidas entre ellos. Estos elementos se organizan 
      como un conjunto de tablas con columnas y filas. Las tablas se utilizan
      para guardar informacion sobre objetos que se van a representar en la
      base de datos.

    * Es una base de datos relacional, cada fila de la tabla es un registro
      con un ID unico llamado clave. Las columnas de la tabla contienen atributos 
      de los datos. y cada registro geberalmente tiene un valor para cada atributo.

    
Vntajas de una base de datos relacional:
   
      la principal ventaja de una base de datos relacional es la sencillez del 
      modelo relacional, permite manejar grandes cantitades de datos con puntos 
      de relacion entre si, gestionandolos de forma segura y conforme a las normas.
      Por ejemplo (Cundo hacemos una tranferencia bancaria y esta se refleja en la 
      aplicacion del banco en nuestro movil de manera inmediata).

      INICIO:Diferencia entre SQL Y NOSQL
    

      ¿Una base de datos relacional vs no relacional?

      ¿Cuando nos comviene usar uno u otro tipo de base de datos?

      Si conocimos de antemano la informacion que necesitamos registrar,
      podremos diseñar la estructura de la base de datos relacional y las 
      tablas que necesitamos para relacionarlas, de manera que sea sencillo 
      y rapido acceder a los datos que queremos consultar en cada momento.
      
      Una base de datos no relacional no tiene un identificador que sirva
      para relacionar un conjunto de datos con otro. Normalmente se emplea
      cuando la información se organiza mediante documentos o cuando 
      no tenemos un esquema exacto de lo que vamos a almacenar. 


      ¿Que pasa cuando tengo gran volumenes de datos que manejar?

      Enfrentamos aqui a la base de datos relacional vs NoSQL o NO Only SQL
      (no solo sql).Estas bases de datos sugirieron con la aparicion de las 
      redes sociales y el incremento de datos que supieron. En principio 
      no usan lenguajes SQL o si no lo hacen, es solo apoyo pero no realizan
      las consultas. Si en bases de datos relacional, los atributos de un 
      elemento están en diferentes columnas. Para realizar las consultas 
      emplean lenguajes propíos como JSON,CQL O GQL y no permiten JOINS

Tipo de datos Númericos:


         Permite almacenar un nuemero. Dependiendo del espacio que
         ocupan y la precision que tiene se puede desglozar en :
         
         ENTEROS:

         Solo permite valores positivos y negativos sin punto decimal.
         De los números son los que ocupan menos espacio y son excelentes
         para guardar datos que no necesitan fracciones. 

         REALES:
        Son números , pero a diferencia de los enteros permite valores fraccionalrios.
        Ocupan mas espacio y se guardan en un formato diferente a los enteros.
        Son utiles cuando el campo a guardar puede contener fracciones décimales.
        (Ejem: Salario,Porcentajes,Promedio).

         NÚMERICO, DECIMAL, PESOS:

         Son otro tipo de datos muy parecidos a los reales por que pueden guardar
         valores décimales pero donde se tiene un poco más de control sobre la precisión.

      
        
Tipo de datos Texto:


        Permiten guardar caracteres de texto, donde un caracter
        hasta textos muy grandes.No tienen formato y se pueden dividir en.

         Textos de longitud fija CHART:

        Cuya longitud siempre sera definida por el campo. Si un campo. Si 
        un campo (nombre) es de longitud 100 por ejemplo y se guarda un (juan)
        solo se usan 4 caracteres, los otros 96 no se ocupan; pero se apartan
        siempre 10 caracteres para guardar el nombre.
        
         Textos de longitud variable (VARCHAR):

         Cuya longitud máxima sera definida en el campo. Es decir el mismo 
         ejemplo del punto anterior que (juan) solo ocupara 4 caracteres 
         y aun que el nombre el nombre se haya definido de longitud 100, los 
         otros 96 caracteres se despediran.

         Textos muy grandes:

         Son capaces de contener un libro completo. Este tipo de datos 
         permite guardar textos muy extensos, pero se pierde en funcionalidad
         de busqueda si lo comparamos con los dos anteriores.
