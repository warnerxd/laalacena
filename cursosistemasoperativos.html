        -cIndica nivel de revision:Cambia cuando hay
        cambios menores como parches de seguridad.

        -dUltimo subnivel de version:Para correciones de fallo


        -Todas las versiones de kernel son publicas y de codigo abierto.

        -El kernel es un archivo grande ya que las actualizaciones se 
        suman al kernel.

        -Eso permite usar linuxen los primeros computadores
        donde fue instalado.

        -Cada actualizacion añade controladores y cambia 
        funciones para optimizar el S.O.

INICIO:SISTEMAS DE ARCHIVOS


            SISTEMA DE ARCHIVOS

        -Permite conocer como es que linux administra los archivos y carpetas.

        -Cuando hablamos de GNU /LInux es muy importante conocer como funciona 
        el sistema de archivos.

        -Linux tiene un sistema muy organizado.

        -El origen de este sistema esta heredado de su predecesor UNIX

        -Todo sistema parte de una raiz (root) "/"

        -Todos los demas archivos se encuentran debajo de esta raiz
        en forma de arbol.


        DIRECTORIO /BIN

        -ES la primera carpeta / directorio que nos encontramos.

        -Contiene los archivos binarios o compilados de los programas basicos del SO.

        -Programas basicos:Navegar en el sistema de archivos,comandos para 
        el tratamiento de permisos,comandos de montaje /demontaje.

        DIRECTORIO /BOOT

        -Contiene todo lo necesario para ejecutar el proceso 
        de arranque de todo el sistema.

        -Almacena todos los datos que se utilizan antes de que
        el kernel ejecute los programas como modo usuario

        -Contiene copias del kernel sel sistema operativo.

        DIRECTORIO /CDROM

        -Existe por la necesidad que habia anteriormente 
        de un directorio donde montar las unidades CD-ROM. 

        -Hoy en dia esta en desuso.

        DIRECTORIO /DEV

        -Aqui se almacena la informacion para que los dispositivos 
        fisicos funcionen.(DRIVERS)


        DIRECTORIO /ETC

        -Almacena archivos adicionales

        -Nunca deben haber archivos binarios

        -Aqui hay dos grandes grupos de archivos: ARCHIVOS Y CARPETAS de 
        configuracion global y ARCHIVOS ESQUELETO:(utilizados con valores por defecto)
        para la configuracion del usuarios del sistema.


        DIRECTORIO /HOME

        -Equivalente a user de WINDOWS

        -Almacena toda la informacion personal

        -Por cada usuario hay  una carpeta /HOME

        DIRECTORIO /ROOT

        -Es la carpeta asignada al usuario adminstrador del sistema

        -Sele conoce como usuario raiz

        -Para acceder a este directorio se deben tener credenciales
        de administrador.


        DIRECTORIO /LIB

        -Guarda bibliotecas de todos los paquetes intalasdos.

        -Esto se debe a que actualmente se usan SO de 64 bist e implica
        que en la raiz del sistema de archivos tengamos un grupo de carpetas 
        llamadas /lib, /lib32 y lib64.

        DIRECTORIO /MEDIA

        -Almacena los dispositivos "montados" En el SO
        -ejm Memoria flash cd,dvd ETC
        -Hoy en dia basta con insertar el dispositivo y automaticamente 
        se monta.Se crea una nueva carpeta dentro de este directorio con 
        referencia del dispositivo.


        DIRECTORIO /MNT

        -Funciones similares a /MEDIA.

        -Se usa muy poco para montar discos temporalmente.

        DIRECTORIO /OPT

        -Aqui se ubican las bibliotecas temporales

        -Inicialmente para almacenar las bibliotecas del 
        software opcional.

        -Su objetivo :NO afectar el sistema de archivos  del so
        en caso de que ocurra algun error en la intalacion o 
        desistalacion. 

        -Cada vez se usa menos

        DIRECTORIO /SBIN

        -Almacena archivos binarios.

        -Adiferencia de /bin , solamente almacena los archivos 
        binarios del sistema.

        -Solo el sistema root puede ejecutarlos.

        DIRECTORIO /USR

        -Contiene la mayor parte de las aplicaciones instaladas del sistema.

        -Entre las aplicaciones mas conocidas que se instalan son:

        -LIbre office 
        -gimp
        -versiones java jdk

        -Controladores de hadware.

        -Tiene su propia jerarquia de carpetas /bin , /lib y /sbin

        -Mantiene la misma filosofia de funcionamiento de que sus equivalentes 
        al mismo nivel de /USR

        -Su subdirectorio  /share contiene todos los iconos del sistema
        que pueden ser admistrados de forma fasil con una interfaz grafica (GUI)


        DIRECTORIO /VAR

        -Almacenan la informacion de los programas de forma temporal

        -Su contenido no debe ser borrado

        -Archivos Logs,temporales,cache copias de seguridad.

        DIRECTORIO /VAR

        -Configuracion del sistema 

        -opt x11 sgml

        BUSQUEDA DE ARCHIVOS Y CARPETAS
        


        find sintaxis

        find [ruta] [opciones] [expresión]

        ruta:el directorio donde comienza la búsqueda. Si no se especifica, find buscará en el directorio actual.
        opciones: parámetros que modifican el comportamiento de la búsqueda.
        expresión: condiciones que deben cumplir los archivos o directorios.


        Ejemplos comunes

          Buscar archivos por nombre:

         find /ruta/a/dir -name "nombre_archivo"


         Buscar directorios por nombre:

         find /ruta/a/dir -type d -name "nombre_directorio"

         Buscar archivos modificados en los últimos n días:

         find /ruta/a/dir -mtime -n

         Buscar archivos más grandes que un tamaño específico:

         find /ruta/a/dir -size +n[cwbkMG]

            c para bytes.
            w para dos bytes.
            b para bloques de 512 bytes.
            k para kilobytes.
            M para megabytes.
            G para gigabytes.
        

        Ejecutar un comando en los archivos encontrados:

        find /ruta/a/dir -name "nombre_archivo" -exec comando {} \;


        file:

            Su finalidad nos indica si un archivo esta vacio o de que clase de contenido esta compuesto.

        1. Editar el archivo de configuración de Netplan

        En Ubuntu Server, los archivos de configuración de red de Netplan suelen estar en /etc/netplan/. 
        Puede haber un archivo de configuración predeterminado con un nombre como 00-installer-config.yaml
        o 01-netcfg.yaml.
        
        Para editar este archivo, usa un editor de texto (como nano):

        sudo nano /etc/netplan/00-installer-config.yaml
            
        2. Configurar la red

        Dentro del archivo YAML, puedes configurar la red, ya sea con una IP estática o mediante DHCP
        (IP asignada automáticamente). Aquí hay ejemplos para cada configuración.
        Configuración de IP dinámica (DHCP)

        Si quieres que tu servidor obtenga la dirección IP automáticamente a través de DHCP, 
        utiliza la siguiente configuración básica:

        network:
            version: 2
            ethernets:
                enp3s0:  # Cambia "enp3s0" por el nombre de tu interfaz de red
                    dhcp4: true
        
                    
        Configuración de IP estática

        Si necesitas una IP fija, puedes configurar la red de esta forma:      
        
        network:
         version: 2
         ethernets:
            enp3s0:  # Cambia "enp3s0" por el nombre de tu interfaz de red
             dhcp4: false
             addresses:
                - 192.168.1.100/24  # Dirección IP y máscara de subred
             gateway4: 192.168.1.1  # Dirección IP de la puerta de enlace
             nameservers:
              addresses:
                - 8.8.8.8  # Servidor DNS (puedes añadir más)
                - 8.8.4.4

        3. Aplicar la configuración de Netplan
    
         sudo netplan apply

        1. Instalación de BIND9

            Primero, instala BIND9 si aún no lo has hecho:

            sudo apt update
            sudo apt install bind9 bind9utils bind9-doc

        2. Configuración del archivo principal: named.conf.options

        Edita el archivo named.conf.options que se encuentra en /etc/bind/:

        sudo nano /etc/bind/named.conf.options

        sudo nano /etc/bind/named.conf.options
        
        3. Configuración de zonas en named.conf.local

        El archivo named.conf.local define las zonas que el servidor va a manejar. Abre el archivo para agregar tus zonas:

        sudo nano /etc/bind/named.conf.local

        Ejemplo de configuración para una zona directa (example.com) y una zona inversa (para una red 192.168.0.x):


        zone "example.com" {
            type master;
            file "/etc/bind/db.example.com";
        };
        
        zone "0.168.192.in-addr.arpa" {
            type master;
            file "/etc/bind/db.192";
        };

        
        4. Crear los archivos de zona

        Zona directa

        Crea el archivo para la zona directa en /etc/bind/db.example.com:

        sudo cp /etc/bind/db.local /etc/bind/db.example.com

        sudo nano /etc/bind/db.example.com   
        

        Ejemplo de configuración:

        $TTL 86400
        @   IN  SOA     ns1.example.com. admin.example.com. (
                        2          ; Serial
                        604800     ; Refresh
                        86400      ; Retry
                        2419200    ; Expire
                        604800 )   ; Negative Cache TTL

        ; Servidores de nombres
        @       IN  NS      ns1.example.com.
        ns1     IN  A       192.168.0.1

        ; Registros A
        @       IN  A       192.168.0.10
        www     IN  A       192.168.0.10

        5. Revisar la configuración y reiniciar BIND9

        Verifica la configuración de BIND9 con:

        sudo named-checkconf

        Verifica los archivos de zona:

        sudo named-checkzone example.com /etc/bind/db.example.com
        sudo named-checkzone 0.168.192.in-addr.arpa /etc/bind/db.192

        Si no hay errores, reinicia el servicio de BIND:

        sudo systemctl restart bind9
